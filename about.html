What is this project?

AccessAble is an independent project created by a web development student to explore current web accessibility standards and to showcase the tools available today for building inclusive experiences.

Who am I?

I’m a web dev student learning how to apply accessibility best practices in real projects. This site is both a learning journal and a practical resource for others.

About me

I’m a 26-year-old woman based in Cyprus, focused on improving my web development skills and building inclusive, accessible interfaces.

This website is a hands-on project to showcase my skills—especially in web accessibility—through semantic HTML, ARIA, keyboard-friendly interactions, and practical tooling.

Skills

  • Semantic HTML and landmarks
  • Accessible forms and labels
  • ARIA basics (roles, properties, states)
  • Keyboard navigation and focus management
  • Color contrast and responsive design
  • Tooling: WAVE, axe DevTools, ANDI

Current learning

  • WCAG 2.2 success criteria
  • Advanced ARIA patterns (menus, dialogs)
  • Screen reader testing (NVDA, VoiceOver)
  • Combining automated and manual audits
  • Design systems with baked-in accessibility

Goals

  • Understand and apply core accessibility standards (WCAG, ARIA, semantic HTML).
  • Collect and compare helpful tools for auditing and improving accessibility.
  • Share concise, actionable resources for developers and content authors.

Standards and references

Key references used while building this project include:

  • WCAG (Web Content Accessibility Guidelines) (opens in new tab)
  • WAI-ARIA 1.2 (opens in new tab)
  • WebAIM articles and checklists (opens in new tab)

This site aims for best practices but is a learning project, not an official standard or certification.
